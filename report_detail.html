2. 핵심 기술: RAG (Retrieval-Augmented Generation)

검색 증강 생성: AI가 답변 시 외부 문서를 검색(Retrieval)하여 컨텍스트로 주입한 뒤 생성(Generation)하는 기술. 일반 ChatGPT는 학습 데이터에만 의존하지만, RAG는 우리 제품 문서만 기반으로 답변하여 환각(Hallucination)을 최소화합니다.

RAG의 3가지 핵심 구성요소

  • Embedding: 텍스트를 1536차원 벡터(숫자 배열)로 변환. 의미가 유사한 텍스트는 벡터 공간에서 가까운 위치에 매핑
  • Vector DB: 벡터를 저장하고 코사인 유사도 기반 ANN(근사 최근접 이웃) 검색. 키워드가 아닌 의미로 검색
  • LLM: 검색된 문서 + 질문을 받아 문서 기반 답변을 생성

4. 청크(Chunk)란?

문서를 AI가 검색하기 좋게 500자씩 잘라놓은 조각입니다. 매뉴얼 100페이지를 통째로 AI에게 주면 비용이 25배, 속도가 10배 느려집니다. 그래서 500자씩 잘라서 저장해놓고, 질문이 들어오면 479개 중 관련 있는 15개만 골라서 AI에게 넘깁니다.

왜 500자인가?

크기장점단점
100자검색 정확문맥 끊김
500자 (현재)검색 정확 + 문맥 유지최적
2000자문맥 충분관련 없는 내용 섞임

500자 = A4 약 1/3 페이지. 한 주제가 담기기에 적당한 크기.

동작 예시: "수신감도 설정법?" 질문 시

청크 #142 (선택) 수신감도는 Echo AMP 메뉴에서 설정. 범위 1~50, 기본값 15...
청크 #87 (선택) 퀵 메뉴에서 EXIT 3초 → 수신감도 바로 조정 가능...
청크 #231 (제외) 릴레이 R1 접점닫힘 조건... → 관련 없음

479개 중 관련 15개만 선택 → 정확하고 빠르고 저렴

6. 소스코드 기반 MD 파일 생성

펌웨어 소스코드의 헤더파일(.h)을 분석하여, 고객이 이해할 수 있는 기술 문서(MD)로 변환했습니다. 소스코드(함수명, 변수명, 코드 스니펫)는 포함하지 않았습니다.

추출 대상

  • 메뉴 구조 (menu_main.h)
  • 측정 파라미터 범위/기본값 (menu_measure.h)
  • 출력/릴레이 설정값 (menu_output.h)
  • 통신 프로토콜 (modbus_master.h)
  • 에러 코드 (app_main.h)
  • 측정 원리/신호 처리 (measure_calc.h, measure_analyze.h)
  • 데이터 저장 (menu_data.h)
  • 시스템 설정 (menu_system.h)

8. AI 프롬프트 엔지니어링

전 제품 공통 프롬프트

  • 문서 간 교차검증 후 답변
  • 문서에 직접 없어도 관련 내용으로 추론 허용
  • 단계별 실무 절차 제공
  • 깊이 생각하고 답변 (Think deeply)
  • 옵션명 명확히 표기 (한글/영문 병기)
  • 애매한 경우 "정확하지 않을 수 있습니다" 표현
  • 문서 간 버전 차이 시 안내
  • 표/그림/챕터 번호 언급 금지
  • 문서에 없으면 고객지원센터(041-584-8820) 안내
  • 제품별 전용 용어 규칙 적용

ENV120 추가 프롬프트 (GPTs 참고)

OpenAI GPTs의 ENV120 프롬프트를 참고하여 추가 규칙을 적용했습니다:

  • 측정 오차 질문 시 수신감도 > Threshold 우선 안내
  • 릴레이 질문 시 R1/R2 접점 닫힘/열림 조건 명확히
  • TVG 용어 사용 제한 (질문에 없으면 안 씀)
  • 메뉴 접근 방법(버튼 순서) + 설정값 동시 안내
  • 현장 권장값/범위 포함
  • 엔지니어 매뉴얼 기반 상세 답변
  • 필드 엔지니어가 바로 적용 가능한 수준

9. 개발 과정

1. 기획 및 설계 — 제품 문서 현황 파악, RAG 아키텍처 설계, LLM 모델 비교(GPT-5.4 mini/nano/풀), 비용 분석, Streamlit Cloud 배포 결정
2. 문서 수집 및 검증 — 매뉴얼(DOCX), 데이터시트, PPT 교육자료 수집. python-docx로 텍스트 추출 테스트, 테이블/단락 구조 확인
3. 펌웨어 소스코드 분석 — C2S-211(ENV200), C1D-330(ENV130), C1S-210(ENV120) 헤더파일에서 파라미터/메뉴구조/측정원리 추출 → MD 변환. #if 0 비활성 기능 제외
4. RAG 파이프라인 구축 — 문서 청킹(500자, 50자 오버랩) → OpenAI Embedding(1536d) → ChromaDB 저장. 제품별 컬렉션 분리(wess_density, wess_interface, wess_interface_120)
5. 챗봇 UI 개발 — Streamlit 기반 채팅 UI. 제품/언어/모델 선택 드롭다운, 스트리밍 응답(write_stream), 대화 기록 20개 유지, 새 대화 버튼
6. 다국어 지원 — 13개 언어별 greeting/caption/placeholder/spinner/unknown 메시지 정의, 언어 전환 시 UI 전체 변경
7. 프롬프트 최적화 — 제품별 용어 규칙 정의, 교차검증/실무추론/깊이생각 지시 추가, GPTs 프롬프트 참고하여 ENV120 고급 프롬프트 적용
8. PPT/엔지니어 매뉴얼 통합 — python-pptx로 PPTX 직접 텍스트 추출 기능 추가, 엔지니어 매뉴얼 포함, 제외 필터 제거
9. 검색 품질 개선 — Top 5 → Top 15 확장. 더 많은 관련 문서를 AI에게 제공하여 답변 정확도/상세도 향상
10. 배포 및 운영 — GitHub push → Streamlit Cloud 자동 배포, Secrets에 API 키 등록, .env/.gitignore 보안 설정
